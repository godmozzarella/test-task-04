# Online Booking

Страница онлайн-бронирования столика в ресторане: форма с валидацией, имитация отправки и экран подтверждения. Тестовое задание для SAVEUR (Junior Frontend).

**Превью:** <тут будет ссылка>

## Стек

- **Next.js 16** (App Router) + **React 19**
- **TypeScript** — типизированы props, state и обработчики
- **SCSS Modules** — изоляция стилей + переменные палитры
- **react-hook-form** — управление формой и валидацией
- **ESLint + Prettier** — линтинг и форматирование

## Запуск локально

```bash
# установить зависимости
npm install

# запустить дев-сервер
npm run dev
```

Открой [http://localhost:3000](http://localhost:3000).

Прочие команды:

```bash
npm run build   # production-сборка
npm start       # запуск собранного приложения
npm run lint    # проверка ESLint
npm run format  # форматирование Prettier
```

## Принятые решения

- **App Router** вместо Pages Router — это актуальный подход Next.js, удобная файловая структура и серверные компоненты по умолчанию.
- **react-hook-form** взял для управления формой: `mode: 'onBlur'` закрывает требование показывать ошибки при потере фокуса, а `isSubmitting` из коробки даёт состояние лоадера на кнопке — меньше ручного стейта.
- **SCSS Modules** — изоляция классов исключает конфликты стилей между
  компонентами, а переменные SCSS централизуют палитру. Не выбрал обычный CSS или Tailwind, так как с SCSS Modules у меня сильно больше опыта и в примере ТЗ используется как раз такой подход.
- **Валидаторы** - функции без зависимости от реакта, легко тестировать и переиспользовать.
- **react-hook-form** - mode: 'onBlur' — закрывает требование ТЗ показывать ошибки при потере фокуса; лоадер на кнопке берётся из встроенного isSubmitting, без ручного стейта загрузки.
- **Константы** - в constants.ts вынесены все значения без как такового объяснения, чтобы было понятно для чего они используются. 
- **Даты** - сравниваются как ISO-строка (YYYY-MM-DD), а не через Date — это исключает баги с часовыми поясами.
- **Время и гости** - через select с фиксированными опциями: значение физически не может быть невалидным.
- **Field** - вынесено в отдельный компонент для удобного переиспользования и избежания дублирования в коде. 
- **Палитра** — в SCSS-переменных (_variables.scss), подключается в модули через относительный @use.
